# Team Ground Rule

1. **정기 회의 운영**  
   주 2회(월/목) 정기 회의를 진행하며, 각 팀원이 이전 회의 이후 수행한 작업과 다음 회의 전까지의 작업 계획을 공유한다.
2. **의사소통 방식**  
   - Zoom: 온라인 및 오프라인 정기 회의를 병행한다. 프로젝트 진행 상황 공유하며 논의가 필요한 부분을 상의한다.
   - GitHub: 소스 코드 버전 관리 및 Pull Request 기반 코드 리뷰를 진행한다.
   - Notion: 프로젝트 문서, 회의록, 개발 일정 및 작업을 관리한다.
   - Slack: 개발 진행 중 발생하는 이슈를 실시간으로 공유하고 신속하게 대응한다.
3. **코드 리뷰 진행**  
   주요 기능 구현 시 최소 1명의 팀원이 코드 리뷰를 진행한 후 메인 브랜치에 병합한다.
4. **문서화**  
   기능 설계, 주요 의사결정 및 시스템 구조를 문서화하여 팀원 간 공유 및 관리한다.
5. **문제 공유 원칙**  
   개발 과정에서 발생한 문제나 기술적 어려움은 팀원과 빠르게 공유하여 함께 해결한다.
6. **R&R 및 작업 범위 조정**  
   프로젝트 진행 중 작업량 불균형이 발생할 경우, 팀원 간 협의를 통해 역할과 작업 범위를 유연하게 재조정한다.
   - 최현서: 프로젝트 총괄 PM, 영상예술대학원생과의 협업 주도, 시계열 데이터 전처리 및 FastAPI 기반 실시간 데이터 스트리밍 서버 구축
   - 이나겸: Unity 기반 실시간 시각화 구현, Boids 알고리즘 설계 및 FastAPI 연동 통신 서버 구축
   - 김나경: 시스템 아키텍처 설계, LSTM 기반 경로 예측 모델 개발, SHAP 기반 XAI(설명 가능한 AI) 데이터 추출 및 최적화
8. **의사결정 원칙**  
   팀 내 의견 불일치 시 다수결보다 충분한 논의를 우선하며, 합의가 어려울 경우 지도교수에게 조언을 구한다.
