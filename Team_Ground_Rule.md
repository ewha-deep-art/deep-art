# Team Ground Rule

1. **정기 회의 운영**  
   주 2회(월/목) 정기 회의를 진행하며, 각 팀원이 이전 회의 이후 수행한 작업과 다음 회의 전까지의 작업 계획을 공유
2. **의사소통 방식**  
   - Zoom: 온라인 및 오프라인 정기 회의를 병행하여 프로젝트 진행 상황 공유
   - GitHub: 소스 코드 버전 관리 및 Pull Request 기반 코드 리뷰 진행
   - Notion: 프로젝트 문서, 회의록, 개발 일정 및 작업 관리
   - Slack: 개발 진행 중 발생하는 이슈를 실시간으로 공유하고 신속하게 대응
3. **코드 리뷰 진행**  
   주요 기능 구현 시 최소 1명의 팀원이 코드 리뷰를 진행한 후 메인 브랜치에 병합
4. **문서화**  
   기능 설계, 주요 의사결정 및 시스템 구조를 문서화하여 팀원 간 공유 및 관리
5. **문제 공유 원칙**  
   개발 과정에서 발생한 문제나 기술적 어려움은 팀원과 빠르게 공유하여 함께 해결
